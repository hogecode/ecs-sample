# ALB リスナーARN参照修正

## 概要
terraform-aws-alb モジュール v9.0へのアップグレードに伴い、出力の参照方法が変更されました。古い出力参照を修正しました。

## 修正内容

### 問題
`terraform/modules/network/alb/outputs.tf` で以下の出力が存在しない `http_tcp_listener_arns` を参照していた：

```terraform
output "public_alb_http_listener_arn" {
  description = "Public ALB HTTP listener ARN"
  value       = try(module.public_alb.http_tcp_listener_arns[0], "")
}

output "private_alb_http_listener_arn" {
  description = "Private ALB HTTP listener ARN"
  value       = try(module.private_alb.http_tcp_listener_arns[0], "")
}
```

### 原因
terraform-aws-modules/alb/aws v9.0での出力構造変更：
- `http_tcp_listener_arns` → `listeners` へ統合
- `http_tcp_listener_ids` → `listeners` へ統合

UPGRADE-9.0.md に以下の記載がある：
```
- `http_tcp_listener_arns` -> replaced with `listeners`
- `http_tcp_listener_ids` -> replaced with `listeners`
```

### 修正後
```terraform
output "public_alb_http_listener_arn" {
  description = "Public ALB HTTP listener ARN"
  value       = try(module.public_alb.listeners["http"].arn, "")
}

output "private_alb_http_listener_arn" {
  description = "Private ALB HTTP listener ARN"
  value       = try(module.private_alb.listeners["http"].arn, "")
}
```

## 影響範囲

### 直接的な影響
- `terraform/modules/network/alb/outputs.tf` - ALB モジュールの出力定義
- `terraform/main.tf` で参照している：
  - `module.alb.public_alb_http_listener_arn`
  - `module.alb.private_alb_http_listener_arn`

### 依存しているリソース
- `terraform/main.tf`内の以下の出力が参照している：
  - `alb_nextjs_listener_arn`
  - `alb_go_listener_arn`

## 検証
- Registry確認: https://registry.terraform.io/modules/terraform-aws-modules/alb/aws/9.17.0?tab=outputs
  - 提供されている出力: `listeners` マップ
  - 各リスナーは `listeners["<key>"].arn` でアクセス可能

## デプロイ
このコミットによって、terraform plan/apply時の出力参照エラーが解決されます。
