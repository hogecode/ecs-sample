
KMS モジュール実装が完了しました。.tfvars で KMS を指定する代わりに、kms モジュールで自動生成するようにできました。

## 実装内容

### 1. **KMS モジュール作成**
新しいモジュール `terraform/modules/security/kms/` を作成し、以下の KMS キーを管理：
- CloudWatch Logs 用
- RDS 用
- S3 用
- SQS 用
- Secrets Manager 用

各キーは自動ローテーション有効で、環境による削除期間を設定可能です。

### 2. **terraform/main.tf 修正**
- KMS モジュールを Phase 2 で追加
- Secrets Manager モジュールで KMS キーを使用
- CloudWatch Monitoring モジュールで KMS キー ID を参照

### 3. **変数設定**
- `enable_kms_encryption`: KMS 有効/無効フラグ（デフォルト: false）
- `kms_deletion_window_days`: KMS キー削除期間（7-30日）

### 4. **環境別設定**
```hcl
# dev.tfvars
enable_kms_encryption = false   # 開発環境はコスト削減のため無効

# staging.tfvars
enable_kms_encryption = true    # ステージング環境は有効

# prod.tfvars
enable_kms_encryption = true    # 本番環境は有効（削除期間30日）
kms_deletion_window_days = 30
```

## 使用方法

開発環境で KMS を有効にしたい場合：
```hcl
# dev.tfvars で
enable_kms_encryption = true
kms_deletion_window_days = 10
```

すると、KMS モジュールが自動的に必要なキーを作成し、CloudWatch Logs、RDS、S3、SQS、Secrets Manager に適用されます。
