# ECS Service にロードバランサー統合を追加

## 概要
ECS Service の定義に `load_balancer` ブロックが不足していたため、ALB（Application Load Balancer）とのトラフィック統合が機能していませんでした。この修正により、ALB から ECS Task へのトラフィック経路が確立されます。

## 修正内容

### 問題
`terraform/modules/compute/ecs/main.tf` で以下のリソースに `load_balancer` ブロックが定義されていませんでした：
- `aws_ecs_service.nextjs`
- `aws_ecs_service.go_server`

これにより：
- ALB のターゲットグループに ECS タスクが登録されない
- ALB からのトラフィックが ECS に到達しない
- ロードバランサーの役割が機能しない

### 修正内容

#### Next.js Service
```terraform
load_balancer {
  target_group_arn = var.nextjs_target_group_arn
  container_name   = "${var.project_name}-nextjs"
  container_port   = var.nextjs_container_port
}
```

#### Go Server Service
```terraform
load_balancer {
  target_group_arn = var.go_server_target_group_arn
  container_name   = "${var.project_name}-go-server"
  container_port   = var.go_server_container_port
}
```

## 影響範囲

### 修正されたファイル
- `terraform/modules/compute/ecs/main.tf`
  - `resource "aws_ecs_service" "nextjs"`
  - `resource "aws_ecs_service" "go_server"`

### 依存するコンポーネント
- **ALB ターゲットグループ**: Next.js 用、Go Server 用
- **ネットワーク**: private_app_subnet, private_api_subnet
- **セキュリティグループ**: nextjs_security_group, go_server_security_group

## デプロイ時の動作

1. **Terraform Apply 実行**：
   - ECS Service が ALB のターゲットグループに自動的に登録
   - ECS Task の起動後、自動的にヘルスチェック開始
   
2. **ヘルスチェック**：
   - Next.js: `GET /` (ポート 3000)
   - Go Server: `GET /health` (ポート 8080)

3. **トラフィックフロー**：
   ```
   Client → ALB → Target Group → ECS Task
   ```

## 関連設定

### ALB リスナー設定
- **Public ALB (HTTP:80)**: Next.js ターゲットグループにフォワード
- **Private ALB (HTTP:8080)**: Go Server ターゲットグループにフォワード

### ターゲットグループ設定
```terraform
nextjs_target_group_arn = "arn:aws:elasticloadbalancing:...:targetgroup/nextjs/..."
go_server_target_group_arn = "arn:aws:elasticloadbalancing:...:targetgroup/go-server/..."
```

## 注意事項

### CODE_DEPLOY との統合
- 現在 `deployment_controller { type = "CODE_DEPLOY" }` を使用しています
- Blue/Green デプロイメント時に CodeDeploy が load_balancer 設定を自動管理します
- Terraform による load_balancer 変更は CodeDeploy のデプロイメント実行時に CodeDeploy が上書きする可能性があります

### テスト手順
```bash
# ALB DNS 名を確認
terraform output public_alb_dns_name

# Next.js へのアクセス確認
curl http://<ALB-DNS>/

# Go Server へのアクセス確認（内部のみ）
curl http://<Private-ALB-DNS>:8080/health
```
