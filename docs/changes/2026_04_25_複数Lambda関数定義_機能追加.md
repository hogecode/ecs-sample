# Lambda関数の複数定義機能の実装

**日付**: 2026年4月25日  
**ステータス**: 完了  
**関連PR**: なし

## 📋 概要

Lambda関数を1つだけでなく、複数定義できるようにTerraform実装を改良しました。JSON設定ファイルを使った動的管理により、新しい関数の追加が容易になります。

## 🎯 課題

従来の実装では、Lambda関数は `terraform/main.tf` で単一の module ブロックとしてハードコードされていました：

```hcl
# 従来: 1つの関数しか定義できない
module "s3_validation_lambda" {
  source = "./modules/lambda"
  # ... 設定
}
```

**問題点：**
- 新しいLambda関数を追加する際、Terraformコードを修正する必要がある
- 関数の管理が複雑になるにつれて、main.tfが肥大化する
- 環境別の設定変更が困難

## ✅ 解決方法

### 1. **設定ファイルベースの管理** (`terraform/lambda_functions.json`)

Lambda関数の定義をJSONファイルに一元化することで、Terraformコードを修正することなく、関数を追加・変更できるようになりました。

```json
{
  "s3_file_validator": {
    "description": "S3ファイル検証関数",
    "source_path": "../lambda",
    "handler": "index.handler",
    "runtime": "nodejs20.x",
    "timeout": 60,
    "memory_size": 256,
    "environment_variables": {
      "MAX_FILE_SIZE_MB": "10"
    },
    "s3_trigger": {
      "enabled": true,
      "bucket_ref": "file_upload_bucket_id",
      "key_prefix": "uploads/",
      "events": ["s3:ObjectCreated:*"]
    },
    "s3_read_policy": true,
    "tags": {
      "Purpose": "FileValidation",
      "Owner": "Platform"
    }
  }
}
```

### 2. **Terraformの動的生成** (`terraform/main.tf`)

`for_each` ループを使用して、JSON設定から複数のLambda関数を動的に作成：

```hcl
# Lambda関数設定を読み込み
locals {
  lambda_functions_config = jsondecode(file(var.lambda_functions_file))
  lambda_functions_resolved = {
    for name, config in local.lambda_functions_config : name => {
      # ... S3バケット参照を解決
    }
  }
}

# for_each で動的にリソース作成
module "lambda_functions" {
  for_each = var.enable_s3_validation_lambda ? local.lambda_functions_resolved : {}
  
  source = "./modules/lambda"
  # ...
}
```

## 📝 変更内容

### 作成ファイル

| ファイル | 説明 |
|---------|------|
| `terraform/lambda_functions.json` | Lambda関数の定義ファイル（複数関数対応） |

### 修正ファイル

| ファイル | 変更内容 |
|---------|---------|
| `terraform/variables.tf` | `lambda_functions` 型変数と `lambda_functions_file` パス変数を追加 |
| `terraform/main.tf` | 単一moduleから `for_each` ベースの動的生成に変更 |
| `terraform/outputs.tf` | Lambda関数情報の出力を追加 |

## 🚀 使用方法

### 新しいLambda関数を追加する

1. **関数のソースコードを作成**
   ```
   lambda/
   ├── index.ts                  # s3_file_validator
   ├── image-processor/
   │   └── handler.ts            # 新しい関数
   └── pdf-converter/
       └── handler.ts            # 別の関数
   ```

2. **`terraform/lambda_functions.json` に追加**
   ```json
   {
     "s3_file_validator": { ... },
     "image_processor": {
       "description": "画像処理関数",
       "source_path": "../lambda/image-processor",
       "handler": "handler.handler",
       "runtime": "nodejs20.x",
       "timeout": 120,
       "memory_size": 512,
       "environment_variables": {
         "OUTPUT_BUCKET": "processed-images"
       },
       "s3_trigger": {
         "enabled": false
       },
       "s3_read_policy": false
     }
   }
   ```

3. **デプロイ**
   ```bash
   cd terraform
   terraform plan
   terraform apply
   ```

## 📊 スケーラビリティ

このアプローチで以下のスケール対応が可能：

- **10個以上のLambda関数**: JSONリストで管理
- **環境別設定**: 環境ごとに異なる `lambda_functions.json` ファイルを使用可能
- **ライブラリ共有**: Lambda層の追加対応も可能
- **カスタムIAMポリシー**: `policy_statements` で柔軟に設定

## 🔧 設定オプション詳細

### 基本設定

| オプション | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| `description` | string | ✅ | Lambda関数の説明 |
| `source_path` | string | ✅ | ソースコードのパス |
| `handler` | string | ✅ | ハンドラー関数名 (例: `index.handler`) |
| `runtime` | string | ✅ | ランタイム (例: `nodejs20.x`, `python3.11`) |
| `timeout` | number | ✅ | タイムアウト（秒） |
| `memory_size` | number | ✅ | メモリ（MB） |

### 環境変数

```json
"environment_variables": {
  "KEY1": "value1",
  "KEY2": "value2"
}
```

### S3トリガー設定（オプション）

```json
"s3_trigger": {
  "enabled": true,
  "bucket_ref": "file_upload_bucket_id",  // モジュール出力参照
  "key_prefix": "uploads/",
  "events": ["s3:ObjectCreated:*", "s3:ObjectRemoved:*"]
}
```

### IAMポリシー

```json
"s3_read_policy": true,  // S3読み取りポリシーを自動生成
"s3_read_policy": false  // カスタムポリシーを使用
```

### タグ

```json
"tags": {
  "Purpose": "DataProcessing",
  "Owner": "DataTeam",
  "CostCenter": "Engineering"
}
```

## 📤 出力値

デプロイ後、以下の出力値が利用可能：

```hcl
# すべてのLambda関数情報
terraform output lambda_functions

# デプロイ概要
terraform output lambda_functions_summary
```

出力例：
```
lambda_functions = {
  "s3_file_validator" = {
    "function_arn" = "arn:aws:lambda:ap-northeast-1:123456789:function:ecs-sample-dev-s3_file_validator"
    "function_name" = "ecs-sample-dev-s3_file_validator"
    "function_invoke_arn" = "arn:aws:lambda:ap-northeast-1:123456789:function:ecs-sample-dev-s3_file_validator"
    "log_group_name" = "/aws/lambda/ecs-sample-dev-s3_file_validator"
    ...
  }
}

lambda_functions_summary = "Deployed 1 Lambda function(s): s3_file_validator"
```

## 🔄 Terraform関数命名規則

動的作成されるLambda関数は、以下の命名規則に従います：

```
{project_name}-{environment}-{function_name_from_json}
```

例：
- `ecs-sample-dev-s3_file_validator`
- `ecs-sample-prod-image_processor`

## ⚙️ トラブルシューティング

### JSON解析エラー

```bash
Error: Failed to decode JSON: ...
```

**対策**: `terraform/lambda_functions.json` のJSON形式を確認
```bash
jq . terraform/lambda_functions.json
```

### S3バケット参照エラー

```bash
Error: Invalid bucket_ref: unknown_bucket
```

**対策**: `bucket_ref` に存在する値を指定
```json
"bucket_ref": "file_upload_bucket_id"  // ✅ 正しい参照
```
